# 迷宫生成系统

> **系统:** Maze Generation
> **重要性:** ⭐ 核心算法
> **设计理念:** 迷宫是游戏的舞台，需要可探索性、随机性、分层复杂度、动态元素

---

## 1. 核心算法：递归回溯 + 房间插入

### 1.1 算法选择对比

| 算法 | 完美迷宫 | 自然感 | 房间支持 | 性能 | 适用性 |
|------|----------|--------|----------|------|--------|
| 递归回溯 | ✓ | ★★★ | 易添加 | O(N) | ⭐⭐⭐⭐⭐ |
| Prim算法 | ✓ | ★★ | 难添加 | O(N log N) | ★★★ |
| Kruskal | ✓ | ★★★ | 难添加 | O(N α(N)) | ★★★ |
| 细胞自动机 | ✗ | ★★★★ | 自然生成 | O(N) | ★★ |
| 分形 | ✓ | ★★★★★ | 难控制 | O(N log N) | ★★ |

**结论:** 递归回溯最适合我们的需求：
- 生成完美迷宫（必有解）
- 易于理解和实现
- 方便添加房间和特殊区域
- 性能优秀

### 1.2 递归回溯算法原理

```
1. 初始化网格，全部为墙
2. 选择起始点（奇数坐标）
3. 标记为通路，压入栈
4. 当栈不为空：
   a. 获取当前位置的未访问邻居（间隔2格）
   b. 如果有邻居：
      - 随机选择一个邻居
      - 打通之间的墙壁
      - 标记邻居为通路
      - 邻居压入栈
   c. 如果没有邻居：
      - 弹出栈顶（回溯）
5. 在最远两点设置入口和出口
```

### 1.3 房间插入算法

```
1. 生成基础迷宫
2. 随机选择位置（不破坏连通性）
3. 创建房间（矩形区域）
4. 打通房间与迷宫的连接
5. 在房间内放置特殊内容
```

## 2. 迷宫层级设计

### 2.1 四层架构

| 层级 | 名称 | 尺寸 | 特点 | 动态变化 |
|------|------|------|------|----------|
| Layer 1 | 表层遗迹 | 21×21 | 教学层，记忆稳定 | 无 |
| Layer 2 | 古代回廊 | 31×31 | 中等难度，开始变化 | 偶尔 |
| Layer 3 | 迷失深渊 | 41×41 | 高难度，快速遗忘 | 经常 |
| Layer 4 | 混沌核心 | 51×51 | 极限挑战，完全动态 | 持续 |

### 2.2 层级特性

#### 表层遗迹 (Layer 1)
- 记忆衰退周期: 30分钟
- 迷宫变化: 无
- 特殊地形: 基础
- 敌人密度: 低
- 资源丰度: 高

#### 古代回廊 (Layer 2)
- 记忆衰退周期: 20分钟
- 迷宫变化: 偶尔（每30分钟）
- 特殊地形: 房间、隐藏门
- 敌人密度: 中
- 资源丰度: 中

#### 迷失深渊 (Layer 3)
- 记忆衰退周期: 10分钟
- 迷宫变化: 经常（每15分钟）
- 特殊地形: 传送点、单向通道
- 敌人密度: 高
- 资源丰度: 低

#### 混沌核心 (Layer 4)
- 记忆衰退周期: 5分钟
- 迷宫变化: 持续（每5分钟）
- 特殊地形: 全部类型
- 敌人密度: 极高
- 资源丰度: 极低

## 3. 特殊地形系统

### 3.1 地形类型

| 类型 | 说明 | 出现层级 |
|------|------|----------|
| **入口** | 返回上层的楼梯 | 所有 |
| **出口** | 前往下层的楼梯 | 所有 |
| **隐藏门** | 需要发现才能通过 | L2+ |
| **单向通道** | 只能单向通过 | L3+ |
| **传送点** | 瞬间移动到其他位置 | L3+ |
| **陷阱** | 触发负面效果 | L2+ |
| **密室** | 包含宝藏的房间 | L2+ |
| **死胡同** | 无出口的区域 | 所有 |

### 3.2 特殊地形生成规则

```yaml
隐藏门:
  生成概率: 5%
  发现难度: 需要仔细检查或特定技能
  奖励: 捷径或宝藏

单向通道:
  生成概率: 3%
  方向: 随机
  风险: 可能进入死路

传送点:
  生成概率: 2%
  成对出现: true
  风险: 可能传送到危险区域

陷阱:
  生成概率: 4%
  类型: [伤害, 困住, 迷惑, 消耗]
  可发现: 高感知或仔细观察
```

## 4. 房间系统

### 4.1 房间类型

| 类型 | 尺寸 | 内容 | 稀有度 |
|------|------|------|--------|
| **储藏室** | 3×3 | 基础物资 | 常见 |
| **宝藏室** | 4×4 | 稀有物品 | 普通 |
| **祭坛** | 5×5 | 恢复/强化 | 稀有 |
| **图书馆** | 6×4 | 古代地图碎片 | 稀有 |
| **陷阱房** | 4×4 | 危险但高回报 | 普通 |
| **安全屋** | 4×4 | 休息恢复 | 普通 |
| **Boss房** | 7×7 | 强敌+丰厚奖励 | 每层1个 |

### 4.2 房间生成规则

```
1. 确定房间数量 (基于迷宫大小)
2. 随机选择位置 (避免重叠)
3. 确保房间与迷宫连通
4. 放置房间内容
5. 添加房间到地图数据
```

## 5. 动态迷宫系统

### 5.1 变化机制

```
迷宫变化触发条件:
1. 时间触发: 每X分钟检查一次
2. 玩家行为: 进入新区域时
3. 特殊事件: 地震、魔法扰动
4. 深层特性: 混沌层持续变化
```

### 5.2 变化类型

| 变化类型 | 说明 | 影响 |
|----------|------|------|
| **墙壁移动** | 通道变墙壁，墙壁变通道 | 改变路径 |
| **房间重组** | 房间位置变化 | 改变布局 |
| **连接改变** | 区域间连接方式变化 | 改变连通性 |
| **新增区域** | 出现之前不存在的区域 | 扩展迷宫 |
| **区域消失** | 某些区域暂时无法进入 | 限制移动 |

### 5.3 变化规则

```
变化约束:
1. 始终保持至少一条通路到出口
2. 不会将玩家困在封闭区域
3. 玩家当前位置不会直接变化
4. 已探索区域变化概率较低
5. 深层变化更频繁、更剧烈
```

## 7. 技术实现

### 7.1 迷宫数据结构

```gdscript
## 迷宫单元格类型
enum CellType {
    WALL = 0,       # 墙壁
    PATH = 1,       # 通道
    ROOM = 2,       # 房间
    ENTRANCE = 3,   # 入口
    EXIT = 4,       # 出口
    SECRET_DOOR = 7,# 隐藏门
    TELEPORTER = 8, # 传送点
    TRAP = 9,       # 陷阱
}

## 迷宫单元格
class MazeCell:
    var x: int
    var y: int
    var type: int
    var discovered: bool = false
    var visible: bool = false

## 迷宫数据
class MazeData:
    var width: int
    var height: int
    var layer: int
    var cells: Array
    var rooms: Array
    var entrance: Vector2i
    var exit: Vector2i
    var seed: String
```

### 7.2 生成器实现

```gdscript
## 迷宫生成器
class_name MazeGenerator

func generate(width: int, height: int, seed: String = "") -> MazeData:
    # 确保奇数尺寸
    var w = width if width % 2 == 1 else width + 1
    var h = height if height % 2 == 1 else height + 1

    # 初始化全为墙
    var maze = _init_maze(w, h)

    # 递归回溯生成
    _recursive_backtrack(maze)

    # 插入房间
    _insert_rooms(maze)

    # 设置入口出口
    _set_entrance_exit(maze)

    return maze

func _recursive_backtrack(maze: MazeData) -> void:
    var stack = []
    var start = Vector2i(1, 1)
    stack.append(start)
    maze.cells[start.y][start.x].type = CellType.PATH

    while stack.size() > 0:
        var current = stack.back()
        var neighbors = _get_unvisited_neighbors(maze, current)

        if neighbors.size() > 0:
            var next = neighbors[randi() % neighbors.size()]
            # 打通墙壁
            var wall = (current + next) / 2
            maze.cells[wall.y][wall.x].type = CellType.PATH
            maze.cells[next.y][next.x].type = CellType.PATH
            stack.append(next)
        else:
            stack.pop_back()
```

# 扩展玩法设计

> 核心玩法完成后，这些内容可以让游戏更加丰富。

---

## 1. Roguelike模式

### 1.1 机制

```
核心循环:
进入迷宫 → 探索绘制 → 收集资源 → 死亡或退出 → 保留部分进度 → 再次挑战
```

### 1.2 永久保留

| 项目 | 保留方式 |
|------|----------|
| 地图知识 | 保留但模糊化（准确度-30%） |
| 等级/技能 | 完全保留 |
| 解锁内容 | 完全保留 |
| 古代地图碎片 | 完全保留 |
| 金币 | 保留50% |
| 物品 | 全部丢失（除装备槽） |

### 1.3 每次变化

- 迷宫完全重新生成
- 物品位置变化
- 敌人配置变化
- 特殊事件随机

## 2. 剧情模式

### 2.1 背景故事

```
《迷雾绘者：失落的文明》

你是一位年轻的地图绘制师，受雇于一个神秘的考古学会。
你的任务是探索最近发现的古代地下迷宫——
据说这里藏有能够"绘制现实"的古代技术。

随着深入，你发现这个迷宫不仅仅是建筑...
它似乎是有生命的，会对你的行为做出反应。
而那些古代地图绘制师，他们到底发现了什么？
为什么他们最后都...消失了？
```

### 2.2 章节设计

| 章节 | 场景 | 核心机制 | 剧情节点 |
|------|------|----------|----------|
| 第一章 | 表层遗迹 | 教学，基础绘制 | 发现古代地图碎片 |
| 第二章 | 古代回廊 | 引入记忆衰退 | 遇到第一个"前辈"的遗骸 |
| 第三章 | 迷失深渊 | 动态迷宫 | 发现迷宫会"学习" |
| 第四章 | 混沌核心 | 全部机制 | 面对古代技术的真相 |
| 终章 | 绘者之心 | 最终挑战 | 选择：使用或封印 |

### 2.3 多结局

- **结局A: 绘者** - 使用技术，成为新的迷宫主宰
- **结局B: 守护者** - 封印技术，永远守护迷宫
- **结局C: 传承者** - 带走知识，继续探索世界
- **隐藏结局: 真相** - 发现一切都是循环

## 3. 多人合作模式

### 3.1 合作探索

```
2-4人组队，共享目标：绘制完整地图并到达出口
```

#### 分工建议

| 角色 | 专精 | 职责 |
|------|------|------|
| 测绘师 | 地图绘制 | 负责主要地图记录 |
| 探险家 | 视野/移动 | 前方探路，发现区域 |
| 资源官 | 背包管理 | 管理光源、食物、工具 |
| 记录员 | 标记系统 | 标记危险、宝藏、路线 |

#### 地图共享

- 各自绘制自己的地图
- 可以交换地图副本
- 副本可能有误差（取决于原地图准确度）
- 可以合并地图（需要找到重叠区域）

### 3.2 竞争模式

```
《绘者竞赛》

多个玩家进入相同迷宫（相同种子）
目标：最快完成地图绘制并到达出口
```

#### 规则

- 迷宫对所有人相同
- 资源有限，先到先得
- 可以干扰对手（封闭通道、误导标记）
- 最终评分：完成时间 × 地图完整度

## 9. 参考设计：《羊蹄山之魂》事件卡组系统

> 参考GDC演讲：《两天回本，首月流水16亿：这群天才，解决了开放世界最大痛点》
> 核心设计：用《龙与地下城》DM发牌的逻辑，代替传统平铺式任务列表

### 9.1 核心概念

**传统开放世界的问题：**
- 任务列表平铺，玩家感到 overwhelmed
- 缺乏节奏控制，内容释放无序
- 探索动力不足，变成"清单式"游玩

**"事件卡组"解决方案：**
```
将任务/事件设计成"卡牌"
DM（游戏系统）根据玩家状态"发牌"
玩家手中的"手牌"决定当前可选内容
```

### 9.2 应用到迷宫游戏

#### 迷宫事件卡组

```typescript
interface EventCard {
  id: string;
  type: EventType;
  priority: number;      // 发牌优先级
  conditions: Condition[]; // 触发条件
  content: EventContent;
  rewards: Reward[];
}

enum EventType {
  DISCOVERY,    // 发现类：隐藏房间、宝藏
  CHALLENGE,    // 挑战类：限时绘制、追逐
  NARRATIVE,    // 叙事类：日记残页、遗迹痕迹
  SOCIAL,       // 社交类：前人标记、幽灵信息
  ENVIRONMENT,  // 环境类：天气变化、地形变动
}
```

#### 发牌逻辑

```
玩家状态检测:
├─ 当前层数
├─ 地图完整度
├─ 体力/压力值
├─ 已探索区域
├─ 持有道具
└─ 游戏时间（昼夜）

发牌策略:
├─ 低压力时 → 发探索/发现类卡牌
├─ 高压力时 → 发安全/恢复类卡牌
├─ 地图完整度高 → 发挑战类卡牌
├─ 长时间未遇敌 → 发危险类卡牌
└─ 特定条件满足 → 发剧情类卡牌
```

### 9.3 5条黄金设计法则（迷宫版）

#### 法则1：玩家永远有选择，但选择有意义

**设计：**
- 手牌上限：3-5张事件卡
- 玩家可以选择接受或跳过
- 跳过有代价（时间流逝、机会成本）

**示例：**
```
手牌:
[发现隐藏门] [遭遇怪物] [发现前人营地]

选择接受"发现隐藏门":
→ 可能获得捷径或宝藏
→ 也可能触发陷阱

选择跳过:
→ 继续探索，但可能错过
→ 时间流逝，记忆衰退
```

#### 法则2：节奏控制：张弛有度

**设计：**
```
探索节奏曲线:
┌─────────────────────────────────────┐
│ 紧张 ↑    ╱╲        ╱╲    ╱╲       │
│      │   ╱  ╲      ╱  ╲  ╱  ╲      │
│      │  ╱    ╲    ╱    ╲╱    ╲     │
│      │ ╱      ╲  ╱              ╲   │
│ 平静 │╱        ╲╱                ╲  │
│      └───────────────────────────────│
│        探索  事件  探索  事件  Boss  │
└─────────────────────────────────────┘
```

**实现：**
- 平静期：发探索/发现类卡牌
- 紧张期：发挑战/危险类卡牌
- 高潮：Boss战或重大发现

#### 法则3：内容释放与玩家状态绑定

**设计：**
| 玩家状态 | 优先发牌类型 | 目的 |
|----------|--------------|------|
| 体力充足 | 探索/挑战类 | 鼓励深入 |
| 体力不足 | 安全/恢复类 | 提供喘息 |
| 压力低 | 发现/叙事类 | 沉浸体验 |
| 压力高 | 安全/幻觉类 | 制造紧张 |
| 地图完整度高 | 隐藏/秘密类 | 奖励探索 |
| 迷路中 | 指引/线索类 | 防止挫败 |

#### 法则4：意外与预期的平衡

**设计：**
```
预期内事件 (70%):
- 标准探索发现
- 常规敌人遭遇
- 资源补给点

意外事件 (30%):
- 稀有宝藏房间
- 特殊NPC遭遇
- 环境突变
- 前人幽灵信息
- 地图悖论事件
```

**"发牌"随机性控制：**
- 使用加权随机，而非纯随机
- 保底机制：连续平淡后必出精彩
- 上限控制：同一类型事件不会连续出现

#### 法则5：叙事与玩法融合

**设计：**
```
每张事件卡都有叙事包装:

[事件卡：墙壁上的符号]
├─ 玩法：发现古老符号，考古学家可解读
├─ 叙事："你注意到墙壁上刻着奇怪的纹路..."
├─ 结果A：解读成功 → "这是前人留下的警告：'不要信任镜子'"
├─ 结果B：解读失败 → "这些符号对你来说毫无意义"
└─ 后续：触发相关事件概率提升
```

### 9.5 与现有系统的结合

| 现有系统 | 事件卡组整合 |
|----------|--------------|
| 地图绘制 | 绘制完成触发"发现"类事件 |
| 心理状态 | 压力值影响发牌类型 |
| 记忆衰退 | 长时间未遇事件触发"回忆"卡 |
| 职业系统 | 不同职业看到不同事件选项 |
| 天气系统 | 天气条件触发环境事件 |
| 经济系统 | 完成事件获得符文石奖励 |

### 9.6 优势总结

1. **解决"清单疲劳"** - 不是任务列表，而是动态手牌
2. **节奏可控** - DM逻辑确保张弛有度
3. **个性化体验** - 根据玩家状态发牌
4. **重玩价值** - 每次发牌组合不同
5. **叙事自然** - 事件与玩法融合

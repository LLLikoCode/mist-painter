# 《迷雾绘者》- 迷宫生成系统

> **系统:** Maze Generation  
> **重要性:** ⭐ 核心算法  
> **设计理念:** 迷宫是游戏的舞台，需要可探索性、随机性、分层复杂度、动态元素

---

## 1. 核心算法：递归回溯 + 房间插入

### 1.1 算法选择对比

| 算法 | 完美迷宫 | 自然感 | 房间支持 | 性能 | 适用性 |
|------|----------|--------|----------|------|--------|
| 递归回溯 | ✓ | ★★★ | 易添加 | O(N) | ⭐⭐⭐⭐⭐ |
| Prim算法 | ✓ | ★★ | 难添加 | O(N log N) | ★★★ |
| Kruskal | ✓ | ★★★ | 难添加 | O(N α(N)) | ★★★ |
| 细胞自动机 | ✗ | ★★★★ | 自然生成 | O(N) | ★★ |
| 分形 | ✓ | ★★★★★ | 难控制 | O(N log N) | ★★ |

**结论:** 递归回溯最适合我们的需求：
- 生成完美迷宫（必有解）
- 易于理解和实现
- 方便添加房间和特殊区域
- 性能优秀

### 1.2 递归回溯算法原理

```
1. 初始化网格，全部为墙
2. 选择起始点（奇数坐标）
3. 标记为通路，压入栈
4. 当栈不为空：
   a. 获取当前位置的未访问邻居（间隔2格）
   b. 如果有邻居：
      - 随机选择一个邻居
      - 打通之间的墙壁
      - 标记邻居为通路
      - 邻居压入栈
   c. 如果没有邻居：
      - 弹出栈顶（回溯）
5. 在最远两点设置入口和出口
```

### 1.3 房间插入算法

```
1. 生成基础迷宫
2. 随机选择位置（不破坏连通性）
3. 创建房间（矩形区域）
4. 打通房间与迷宫的连接
5. 在房间内放置特殊内容
```

## 2. 迷宫层级设计

### 2.1 四层架构

| 层级 | 名称 | 尺寸 | 特点 | 动态变化 |
|------|------|------|------|----------|
| Layer 1 | 表层遗迹 | 21×21 | 教学层，记忆稳定 | 无 |
| Layer 2 | 古代回廊 | 31×31 | 中等难度，开始变化 | 偶尔 |
| Layer 3 | 迷失深渊 | 41×41 | 高难度，快速遗忘 | 经常 |
| Layer 4 | 混沌核心 | 51×51 | 极限挑战，完全动态 | 持续 |

### 2.2 层级特性

#### 表层遗迹 (Layer 1)
- 记忆衰退周期: 30分钟
- 迷宫变化: 无
- 特殊地形: 基础
- 敌人密度: 低
- 资源丰度: 高

#### 古代回廊 (Layer 2)
- 记忆衰退周期: 20分钟
- 迷宫变化: 偶尔（每30分钟）
- 特殊地形: 房间、隐藏门
- 敌人密度: 中
- 资源丰度: 中

#### 迷失深渊 (Layer 3)
- 记忆衰退周期: 10分钟
- 迷宫变化: 经常（每15分钟）
- 特殊地形: 传送点、单向通道
- 敌人密度: 高
- 资源丰度: 低

#### 混沌核心 (Layer 4)
- 记忆衰退周期: 5分钟
- 迷宫变化: 持续（每5分钟）
- 特殊地形: 全部类型
- 敌人密度: 极高
- 资源丰度: 极低

## 3. 特殊地形系统

### 3.1 地形类型

| 类型 | 说明 | 出现层级 |
|------|------|----------|
| **入口** | 返回上层的楼梯 | 所有 |
| **出口** | 前往下层的楼梯 | 所有 |
| **隐藏门** | 需要发现才能通过 | L2+ |
| **单向通道** | 只能单向通过 | L3+ |
| **传送点** | 瞬间移动到其他位置 | L3+ |
| **陷阱** | 触发负面效果 | L2+ |
| **密室** | 包含宝藏的房间 | L2+ |
| **死胡同** | 无出口的区域 | 所有 |

### 3.2 特殊地形生成规则

```yaml
隐藏门:
  生成概率: 5%
  发现难度: 需要仔细检查或特定技能
  奖励: 捷径或宝藏

单向通道:
  生成概率: 3%
  方向: 随机
  风险: 可能进入死路

传送点:
  生成概率: 2%
  成对出现: true
  风险: 可能传送到危险区域

陷阱:
  生成概率: 4%
  类型: [伤害, 困住, 迷惑, 消耗]
  可发现: 高感知或仔细观察
```

## 4. 动态迷宫系统

### 4.1 变化机制

```
迷宫变化触发条件:
1. 时间触发: 每X分钟检查一次
2. 玩家行为: 进入新区域时
3. 特殊事件: 地震、魔法扰动
4. 深层特性: 混沌层持续变化
```

### 4.2 变化类型

| 变化类型 | 说明 | 影响 |
|----------|------|------|
| **墙壁移动** | 通道变墙壁，墙壁变通道 | 改变路径 |
| **房间重组** | 房间位置变化 | 改变布局 |
| **连接改变** | 区域间连接方式变化 | 改变连通性 |
| **新增区域** | 出现之前不存在的区域 | 扩展迷宫 |
| **区域消失** | 某些区域暂时无法进入 | 限制移动 |

### 4.3 变化规则

```
变化约束:
1. 始终保持至少一条通路到出口
2. 不会将玩家困在封闭区域
3. 玩家当前位置不会直接变化
4. 已探索区域变化概率较低
5. 深层变化更频繁、更剧烈
```

## 5. 房间系统

### 5.1 房间类型

| 类型 | 尺寸 | 内容 | 稀有度 |
|------|------|------|--------|
| **储藏室** | 3×3 | 基础物资 | 常见 |
| **宝藏室** | 4×4 | 稀有物品 | 普通 |
| **祭坛** | 5×5 | 恢复/强化 | 稀有 |
| **图书馆** | 6×4 | 古代地图碎片 | 稀有 |
| **陷阱房** | 4×4 | 危险但高回报 | 普通 |
| **安全屋** | 4×4 | 休息恢复 | 普通 |
| **Boss房** | 7×7 | 强敌+丰厚奖励 | 每层1个 |

### 5.2 房间生成规则

```
1. 确定房间数量 (基于迷宫大小)
2. 随机选择位置 (避免重叠)
3. 确保房间与迷宫连通
4. 放置房间内容
5. 添加房间到地图数据
```

## 6. 环境叙事元素

### 6.1 遗迹痕迹

**古老符号:**
- 墙壁上随机出现的古代文字
- 考古学家职业可以解读
- 解读成功获得提示（出口方向、危险警告）

**前人标记:**
```
标记类型:
- 箭头标记 → 方向指示（可能正确也可能误导）
- 骷髅标记 ☠ 危险警告
- 问号标记 ❓ 未探索
- 叉号标记 ✗ 死胡同
- 特殊符号 需要解读
```

**可信度机制:**
- 不同前人的标记可信度不同
- 某些是故意误导
- 通过考古学技能判断真伪

### 6.2 日记残页

**收集系统:**
- 散落在迷宫各处
- 逐步揭示迷宫历史和背景故事
- 收集完整可解锁隐藏区域

**内容示例:**
```
[第1页] "我们以为这只是普通的遗迹..."
[第5页] "迷宫在变化，我发誓那条路昨天不在那里"
[第12页] "我看到了他们，那些先我们一步的人，但他们已经不..."
```

### 6.3 动态环境

**周期性变化:**
- **潮汐区域** - 某些通道定时被水淹没/露出
- **机关重置** - 每游戏日重置一次
- **月相影响** - 特定月相时隐藏区域开放

**天气系统:**
- 露天区域受天气影响
- 暴雨改变地形
- 大雾进一步限制视野

## 7. 天气与昼夜系统 (v1.2)

### 7.1 时间流逝

```
1现实分钟 = 10游戏分钟
游戏内时间持续流动
```

**时间影响:**

| 时段 | 效果 |
|------|------|
| 06:00-18:00 白天 | 标准状态 |
| 18:00-20:00 黄昏 | 视野-1，某些敌人活跃 |
| 20:00-06:00 夜晚 | 视野-2，夜间秘密显现 |
| 00:00-02:00 深夜 | 超自然现象概率+50% |

### 7.2 天气效果

| 天气 | 效果 |
|------|------|
| **迷雾** | 视野-50%，但怪物也看不清你 |
| **暴雨** | 水淹区域变化，火把易灭 |
| **地震** | 随机改变部分迷宫结构 |
| **月蚀** | 超自然现象频发，危险与机遇 |
| **血月** | 敌人狂暴，但宝藏翻倍 |

## 8. 地图验证机制 (v1.1)

### 8.1 矛盾检测

**检测类型:**

```typescript
enum MapContradiction {
  WALL_PATH_MISMATCH,    // 画成通道实际有墙
  CONNECTIVITY_ERROR,    // 连通性错误
  DISTANCE_MISMATCH,     // 距离不符
  IMPOSSIBLE_GEOMETRY,   // 几何不可能
}
```

**触发条件:**
- 玩家尝试穿过自己画的"通道"被墙挡住
- 两个已探索区域无法按地图连通
- 测量距离与地图标记不符

### 8.2 修正成本

| 修正方式 | 成本 | 限制 |
|----------|------|------|
| 铅笔修改 | 橡皮耐久-1 | 仅铅笔绘制可改 |
| 墨水覆盖 | 无法覆盖 | 必须重绘 |
| 贴修正纸 | 纸张-1 | 仅限粗糙纸 |
| 重绘区域 | 时间+体力 | 最彻底但成本高 |

### 8.3 "地图混乱"状态

当矛盾积累过多时触发:
- 地图界面出现视觉扭曲
- 无法信任自己的地图
- 必须找到参考点重新校准
- 可能触发特殊剧情

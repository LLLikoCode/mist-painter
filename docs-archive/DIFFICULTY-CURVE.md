## 1. 设计原则与理念

### 1.1 核心设计目标

迷雾绘者作为冒险解谜游戏，难度曲线设计遵循以下原则：

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **渐进式学习** | 玩家通过游戏自然学习机制 | 每层引入1-2个新元素 |
| **可控挑战** | 难度在玩家能力边界附近波动 | 动态调整+手动选择 |
| **正向反馈** | 克服困难后获得显著成就感 | 解锁新能力+视觉反馈 |
| **无惩罚性** | 失败是学习机会而非惩罚 | 进度保留+提示系统 |

### 1.2 难度曲线模型

采用**波浪式渐进模型**，而非线性增长：

```
难度
 100% │                                          ╭──────╮
      │                                    ╭────╯      │
  80% │                              ╭────╯             │
      │                        ╭────╯                  │
  60% │                  ╭────╯                        │
      │            ╭────╯                              │
  40% │      ╭────╯                                    │
      │╭────╯                                          │
  20% │╯                                                ╰────
      │
   0% └────────────────────────────────────────────────────▶
       教学   第1层   第2层   第3层   第4层   第5层   Boss
       关卡   (简单)  (普通)  (困难)  (困难+) (极难)  战
```

**设计要点**：
- 每个层级内部：开头简单 → 中间挑战 → 结尾放松（Boss前准备）
- 层与层之间：前3层快速爬坡，后2层保持高位挑战
- Boss战：难度峰值，综合考验所有 learned skills

## 3. 谜题复杂度递进曲线

### 3.1 谜题类型分级

| 等级 | 类型 | 描述 | 示例 |
|------|------|------|------|
| L1 | **基础揭示** | 单步操作，直接结果 | 绘制简单路径驱散迷雾 |
| L2 | **顺序操作** | 多步顺序执行 | 按顺序点亮符文 |
| L3 | **条件判断** | 根据状态选择操作 | 根据迷雾类型选择画笔 |
| L4 | **空间推理** | 理解空间关系 | 镜像对称绘制 |
| L5 | **时序控制** | 把握时机 | 在特定时机绘制 |
| L6 | **资源优化** | 有限资源下的最优解 | 最少墨水完成绘制 |
| L7 | **复合谜题** | 多种机制组合 | 顺序+条件+空间综合 |

### 3.3 谜题难度递进公式

**复杂度评分公式**：
```
谜题难度分 = 基础分 × 机制数量系数 × 约束条件系数 × 时间压力系数

其中:
- 基础分 = 10 (L1) / 20 (L2) / 35 (L3) / 55 (L4) / 80 (L5) / 110 (L6) / 150 (L7)
- 机制数量系数 = 1 + (机制数-1) × 0.3
- 约束条件系数 = 1 + 约束数 × 0.25
- 时间压力系数 = 1 (无限制) / 1.2 (宽松) / 1.5 (标准) / 2.0 (严格)
```

### 3.4 典型谜题示例

**第1层 - 基础路径绘制 (L1-L2)**
```
目标: 从A点到达B点
机制: 绘制路径驱散迷雾
约束: 路径不能交叉
难度: 15分
```

**第3层 - 符文激活 (L3-L4)**
```
目标: 激活所有符文
机制: 
  - 不同颜色符文需要对应颜色画笔
  - 符文之间有连接线，必须按顺序激活
  - 某些符文被迷雾遮挡，需要先揭示
约束: 墨水有限，顺序错误会重置
难度: 75分
```

**第5层 - 时空绘卷 (L6-L7)**
```
目标: 完成复杂绘卷
机制:
  - 镜像对称绘制（左右同步）
  - 时序触发（在特定时机绘制特定部分）
  - 资源管理（墨水、时间双约束）
  - 动态迷雾（绘制后迷雾会重新生成）
约束: 多失败条件，容错率低
难度: 180分
```

### 4.4 能力-挑战平衡公式

**平衡系数公式**：
```
平衡系数 = 玩家能力值 / 关卡难度值

理想范围: 0.8 - 1.2 (略有挑战但可完成)

- 0.8-1.0: 有挑战性
- 1.0-1.2: 舒适区

```

## 5. 难度自适应调整策略

### 5.1 动态难度调整(DDA)系统

**监测指标**：

| 指标 | 说明 | 权重 | 数据来源 |
|------|------|------|----------|
| 死亡次数 | 玩家失败频率 | 30% | 游戏事件 |
| 谜题耗时 | 解谜所用时间 | 25% | 计时器 |
| 提示使用 | 求助频率 | 20% | 交互记录 |
| 资源消耗 | 墨水/道具使用效率 | 15% | 资源管理 |
| 重试次数 | 同一谜题尝试次数 | 10% | 游戏事件 |

**调整触发条件**：

```
连续挫败检测:
- 3次死亡/10分钟内 → 触发轻度调整
- 5次死亡/同一谜题 → 触发中度调整
- 10分钟无进展 → 触发提示系统
- 连续使用3次提示 → 提供简化方案

连续顺利检测:
- 无伤通过5个房间 → 增加精英敌人
- 提前30%时间完成 → 减少下阶段提示
- 资源剩余>50% → 增加资源消耗点
```

### 5.2 调整措施分级

**轻度调整**（玩家感受不明显）：
- 增加墨水恢复速度 +20%
- 延长谜题时间限制 +15%
- 降低敌人遭遇率 -10%
- 增加存档点密度

**中度调整**（玩家可感知）：
- 简化谜题（减少步骤/约束）
- 提供额外提示
- 降低迷雾覆盖度 -10%
- 增加视野范围 +1格

**重度调整**（明显帮助）：
- 提供"跳过关卡"选项
- 显示完整解法演示
- 临时无敌模式
- 直接降低难度等级

**手动难度调整**：

```

┌─────────────────────────────────────────┐

```

**自适应强度级别**：
- **关闭**: 固定难度，无调整
- **温和**: 仅轻度调整，保持挑战
- **标准**: 轻中度调整，平衡体验
- **积极**: 所有级别调整，确保通关

### 5.4 难度调整算法

```typescript
// 伪代码示例
function calculateDifficultyAdjustment(playerMetrics: Metrics): Adjustment {
  const frustrationScore = 
    playerMetrics.deathCount * 0.3 +
    playerMetrics.puzzleTimeRatio * 0.25 +
    playerMetrics.hintUsage * 0.2 +
    playerMetrics.resourceEfficiency * 0.15 +
    playerMetrics.retryCount * 0.1;
  
  if (frustrationScore > 0.8) {
    return { type: 'HEAVY', value: 0.7 }; // 降低30%难度
  } else if (frustrationScore > 0.6) {
    return { type: 'MEDIUM', value: 0.85 }; // 降低15%难度
  } else if (frustrationScore > 0.4) {
    return { type: 'LIGHT', value: 0.92 }; // 降低8%难度
  } else if (frustrationScore < 0.1) {
    return { type: 'INCREASE', value: 1.1 }; // 增加10%难度
  }
  
  return { type: 'NONE', value: 1.0 };
}
```

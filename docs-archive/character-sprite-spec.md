# 迷雾绘者 (Mist Painter) - 角色Sprite规格说明

> **版本**: v1.0  
> **创建时间**: 2026-03-23  
> **文档类型**: 美术资源规格文档

---

## 1. 概述

本文档定义了《迷雾绘者》主角角色的Sprite资源规格，包括尺寸、动画帧数、配色方案等技术参数。

---

## 2. Sprite规格

### 2.1 基础参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 单帧尺寸 | 32×48 像素 | 标准角色尺寸 |
| 色彩深度 | 32-bit RGBA | 支持透明通道 |
| 文件格式 | PNG | 无损压缩，透明背景 |
| 像素风格 | 像素艺术 | 符合项目美术风格 |

### 2.2 角色设计

#### 视觉特征
- **形象**: 神秘探险者，身穿深色斗篷
- **面部**: 兜帽遮住大部分面部，仅露金色眼睛
- **配色**: 深色调为主，便于在浅色地图背景中识别

#### 配色方案
| 部位 | 颜色值 | 用途 |
|------|--------|------|
| 斗篷主体 | `#2d2d2d` | 深灰色斗篷 |
| 斗篷内部 | `#1a1a1a` | 墨黑色阴影 |
| 兜帽内部 | `#141414` | 更深的阴影 |
| 眼睛 | `#c9a227` | 金色微光 |
| 背包 | `#5a4a3a` | 皮革棕色 |
| 靴子 | `#3d2817` | 深褐色 |
| 高光 | `#3c3c3c` | 边缘高光 |

## 3. 动画规格

### 3.1 待机动画 (Idle)

| 参数 | 值 |
|------|-----|
| 帧数 | 4帧 |
| 帧率 | 4 FPS (每帧250ms) |
| 循环 | 是 |
| 文件 | `character_idle.png` (128×48) |

#### 动画描述
- 轻微呼吸起伏效果
- 斗篷随呼吸轻微摆动
- 整体姿态保持静止

#### 帧序列
| 帧 | 描述 |
|----|------|
| 1 | 正常姿态 |
| 2 | 轻微下沉 (呼吸) |
| 3 | 恢复正常 |
| 4 | 轻微下沉 (呼吸) |

### 3.2 行走动画 (Walk)

| 参数 | 值 |
|------|-----|
| 帧数 | 8帧 |
| 帧率 | 12 FPS (每帧83ms) |
| 循环 | 是 |
| 文件 | `character_walk.png` (256×48) |

#### 动画描述
- 左右腿交替前进
- 斗篷随步伐飘动
- 背包随身体摆动

#### 帧序列
| 帧 | 描述 |
|----|------|
| 1 | 左腿前，右腿后，斗篷左飘 |
| 2 | 左腿前 (延续) |
| 3 | 双腿并拢，中间姿态 |
| 4 | 双腿并拢 (延续) |
| 5 | 右腿前，左腿后，斗篷右飘 |
| 6 | 右腿前 (延续) |
| 7 | 双腿并拢，中间姿态 |
| 8 | 双腿并拢 (延续) |

### 3.3 交互动画 (Interact)

| 参数 | 值 |
|------|-----|
| 帧数 | 4帧 |
| 帧率 | 8 FPS (每帧125ms) |
| 循环 | 否 (单次播放) |
| 文件 | `character_interact.png` (128×48) |

#### 动画描述
- 角色伸手进行交互动作
- 适用于开门、拾取物品等交互
- 播放完毕后返回待机状态

#### 帧序列
| 帧 | 描述 |
|----|------|
| 1 | 准备姿态 (正常待机) |
| 2 | 伸手 |
| 3 | 接触 (手臂完全伸出) |
| 4 | 收回 |

## 5. Godot导入配置

### 5.1 Sprite Sheet导入设置

在Godot中导入Sprite Sheet时，建议以下设置：

```
Import Settings:
- Filter: Nearest (保持像素清晰)
- Repeat: Disabled
- Mipmaps: Disabled (像素艺术不需要)
```

### 5.2 动画播放配置

参考 `character_animation_reference.gd` 文件中的配置代码。

## 6. 技术注意事项

### 6.1 性能优化
- 使用Sprite Sheet减少Draw Call
- 共享相同的材质和纹理
- 动画帧率可根据性能调整

### 6.2 兼容性
- 所有资源兼容Godot 4.x
- 标准PNG格式，跨平台支持
- 透明背景确保正确叠加

### 6.3 扩展性
- 预留了添加新动画的空间
- 单帧文件便于单独修改
- 脚本可重新生成所有资源

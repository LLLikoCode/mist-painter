# 谜题编辑器设计文档

## 1. 概述

### 1.1 设计目标

迷雾绘者谜题编辑器是一个可视化关卡设计工具，旨在让关卡设计师能够：

- 快速创建和编辑游戏关卡
- 可视化配置谜题元素
- 实时预览迷雾效果
- 导出与游戏系统兼容的关卡数据

### 1.2 核心原则

- **直观易用**：拖拽式操作，所见即所得
- **实时预览**：编辑时即时看到效果
- **数据兼容**：导出格式与游戏系统完全兼容
- **模块化设计**：各功能模块独立可扩展

### 2.2 核心模块

#### 2.2.1 LevelEditor（主编辑器）

- **职责**：整合所有子编辑器，管理关卡数据
- **功能**：
  - 初始化Phaser游戏实例
  - 协调地图、谜题、迷雾编辑器
  - 处理用户输入事件
  - 管理关卡数据的加载和保存

#### 2.2.2 MapEditor（地图编辑器）

- **职责**：地形层的编辑
- **功能**：
  - 放置和删除图块
  - 多图层管理（地板/墙壁/装饰）
  - 区域填充
  - 图块选择

#### 2.2.3 PuzzleEditor（谜题编辑器）

- **职责**：谜题元素的配置
- **功能**：
  - 放置各类谜题
  - 编辑谜题属性
  - 设置谜题依赖关系
  - 可视化谜题连接

#### 2.2.4 MistEditor（迷雾编辑器）

- **职责**：迷雾区域的绘制
- **功能**：
  - 笔刷式迷雾清除
  - 迷雾区域标记
  - 迷雾密度调整
  - 快速填充/清除

#### 2.2.5 LevelExporter（导出器）

- **职责**：关卡数据导出
- **功能**：
  - JSON格式导出
  - Godot TSCN格式导出
  - 数据验证
  - 元数据管理

## 3. 交互流程

### 3.1 创建新关卡

```
用户点击"新建" → 创建空关卡数据 → 初始化编辑器 → 显示空白画布
```

### 3.2 编辑地形

```
选择地形工具 → 选择图块类型 → 选择图层 → 在画布点击 → 放置图块
```

### 3.3 放置谜题

```
选择谜题工具 → 选择谜题类型 → 在画布点击 → 创建谜题 → 编辑属性
```

### 3.4 绘制迷雾

```
选择迷雾工具 → 调整笔刷大小 → 在画布拖动 → 清除迷雾区域
```

### 3.5 导出关卡

```
点击导出 → 选择格式 → 验证数据 → 生成文件 → 下载/保存
```

## 4. 数据格式

### 4.1 关卡数据结构

```typescript
interface LevelData {
  metadata: LevelMetadata;      // 关卡元数据
  gridSize: GridSize;           // 网格尺寸
  tileSize: number;             // 图块大小
  layers: {                     // 地形层
    floor: TileData[];
    walls: TileData[];
    decorations: TileData[];
  };
  puzzles: PuzzleData[];        // 谜题数据
  mist: MistData;               // 迷雾数据
  spawnPoint: PixelPosition;    // 出生点
  goal: {                       // 终点
    position: PixelPosition;
    radius: number;
  };
}
```

### 4.2 导出格式

#### JSON格式

适合程序化读取，包含完整关卡数据：

```json
{
  "level_id": "level_01",
  "level_name": "第一关",
  "grid_size": { "width": 40, "height": 22 },
  "tile_size": 32,
  "layers": { ... },
  "puzzles": [ ... ],
  "mist": { ... },
  "spawn_point": { "x": 100, "y": 360 },
  "goal": { "position": { "x": 1100, "y": 360 }, "radius": 40 }
}
```

#### TSCN格式

Godot引擎场景文件，可直接导入项目：

```
[gd_scene load_steps=10 format=3]
[node name="Level01" type="Node2D"]
...
```

## 5. 技术实现

### 5.1 技术栈

- **Phaser 3**：游戏渲染引擎
- **TypeScript**：类型安全
- **Vite**：构建工具
- **原生ES Modules**：模块化

### 5.2 渲染层级

```
Z-Index:
  0: 网格
  1: 地板层
  2: 墙壁层
  3: 装饰层
 10: 谜题层
 20: 迷雾层
 50: 笔刷预览
100: 选中高亮
```

### 5.3 性能优化

- 脏矩形更新：只更新变化区域
- 笔刷缓存：预计算常用笔刷
- 分层渲染：各层独立更新
- 异步计算：覆盖率计算不阻塞

## 6. 扩展性设计

### 6.1 新增谜题类型

1. 在 `PUZZLE_TEMPLATES` 添加配置
2. 在 `PuzzleEditor.createPuzzleVisual` 添加渲染逻辑
3. 在 `LevelExporter` 添加导出逻辑

### 6.2 新增图块类型

1. 在 `TILE_CONFIGS` 添加配置
2. 创建对应纹理
3. 在 `MapEditor` 添加处理逻辑

### 6.3 新增导出格式

1. 在 `LevelExporter` 添加导出方法
2. 在UI中添加格式选项

## 7. 使用说明

### 7.1 快捷键

| 快捷键 | 功能 |
|--------|------|
| 1 | 选择工具 |
| 2 | 地形工具 |
| 3 | 谜题工具 |
| 4 | 迷雾工具 |
| 5 | 橡皮擦 |
| Ctrl+S | 保存 |
| Ctrl+O | 打开 |
| Ctrl+E | 导出 |
| Delete | 删除选中 |
| Escape | 取消选择 |

### 7.2 操作流程

1. **创建关卡**：点击"新建"或使用快捷键
2. **设计地形**：选择地形工具，选择图块，在画布绘制
3. **放置谜题**：选择谜题工具，选择类型，点击放置
4. **配置迷雾**：选择迷雾工具，调整笔刷，绘制清除区域
5. **导出关卡**：点击"导出"，选择格式，保存文件
